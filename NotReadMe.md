## Техническая спецификация проекта

### Основная ценность: Приложение позволяет группе людей в одной локации совместно управлять музыкальным фоном. Хост (компьютер с колонками) выступает центральным узлом воспроизведения, а гости (смартфоны и/или другие ПК/Ноутбуки) — пультом управления через браузер.
## Сценарии использования (User Flows)
Сценарий А: «Вечеринка / Открытая демократия» (Default Mode)
Контекст: Домашняя вечеринка или бар. Нужно, чтобы музыка нравилась большинству.
Действия:
Хост (DJ): Создает комнату «Party», получает QR-код. Подключает ПК к Wi-Fi и интернету. Запускает плеер на большом экране/колонках.
Гость: Сканирует QR, попадает в интерфейс комнаты.
Добавление трека: Гость ищет трек через поиск (VK Music API) и нажимает «Предложить».
Голосование (Интерактив): Трек попадает в очередь со статусом «Кандидат». Другие гости видят его и могут нажать на условные "за" или "против".
Механика: У каждого пользователя есть ограниченный запас «энергии» (например, 5 голосов в час), чтобы избежать спама.
Ротация: Алгоритм автоматически перемещает треки с highest rating в начало очереди воспроизведения.
Воспроизведение: Трек играет только на устройстве Хоста.
Сценарий Б: «Школьная дискотека / Модерация» (Admin Mode)
Контекст: Мероприятие, где есть ответственный за контент (администратор/диджей).
Действия:
Хост (Admin): Создает комнату, загружает «Базовый плейлист» (approved tracks). Включает режим «Модерация».
Гость: Может предложить трек, но он не попадает в общую очередь сразу. Он попадает в панель «Запросы к Админу».
Админ: Видит список запросов на своем экране. Может одобрить (трек летит в очередь) или отклонить.
Приоритет: Гости могут голосовать за то, какой из уже одобренных треков должен сыграть следующим (переупорядочивание очереди).
Технологический стек
1.	Frontend:
o	Framework: React (Vite для быстрой сборки).
o	Real-time: Socket.io-client (для мгновенного обновления очереди и синхронизации статуса плеера).
o	UI Kit: Tailwind CSS + Headless UI (или Mantine) для быстрого верстания адаптивных интерфейсов.
o	QR Generation: qrcode.react.
2.	Backend:
o	Framework: Django + Django REST Framework (DRF).
o	Async/Real-time: Django Channels (необходим для WebSocket соединений, чтобы хост и гости видели изменения очереди в реальном времени без перезагрузки).
3.	Database:
o	PostgreSQL: Хранение пользователей (если нужна авторизация), сессий (комнат), истории голосований, структуры плейлистов.
4.	Media Storage & VK Integration:
o	VK Music API: Используется как источник аудио-потоков и метаданных (обложки, названия).
o	Krutoy-pirat database – современная отечественная разработка на основе технологий zaycev.net, utorrent и др. (для тестов)
5.	Infrastructure:
o	Nginx: Reverse proxy. Раздает статику (React build) и проксирует запросы к Django.
o	Docker: Для контейнеризации (Django, DB, Nginx).

Концепт
Краткий концепт 
Суть: Веб-приложение для совместного создания плейлистов на вечеринках. Гости подключаются через смартфоны, добавляют треки из VK и голосуют за очередь.
Ключевые функции:
 	Голосование за треки (поднимает в очереди)
Режим DJ (управление воспроизведением)
QR-код для быстрого подключения гостей
Поиск треков через VK API
