# WebProject
# Уровень 1: Критический функционал (Ядро системы)

## 1. Хранение и управление медиа-библиотекой (Backend & Admin)

Так как музыка загружается вручную и хранится в собственной БД, это фундамент всего проекта.

**Функционал:**

Загрузка аудиофайлов формата MP3 в защищенное хранилище сервера.

Мета-данные треков: название, исполнитель, альбом, год выпуска, длительность, обложка.

Присвоение системных тегов (жанр, настроение, темп) администратором при загрузке.

Проверка целостности файлов (защита от битых файлов).

Нюанс: Реализация надежной файловой системы, так как объем данных будет расти только за счет действий админа.

## 2. Административная панель (Admin Panel)

Ключевой элемент из-за ручной загрузки и необходимости модерации пиратского контента.

**Функционал:**

Интерфейс для загрузки новых треков/альбомов.

Управление тегами и категориями (создание/редактирование списков жанров и настроений).

Просмотр статистики загрузок и активности пользователей.

Инструменты для бана пользователей или удаления нелегального контента (если поступят жалобы, несмотря на внутреннюю политику).

 ## 3.Аудио-плеер (Client-side)

Основная функция для конечного пользователя.

**Функционал:**

Воспроизведение MP3 (Play/Pause, переключение треков, перемотка).

Отображение текущей композиции и обложки.

Базовое управление громкостью.

Работа в фоновом режиме (для мобильных устройств).

Буферизация потока для минимизации задержек.

## 4. Система авторизации и профили пользователей

Необходима для сохранения личных данных и плейлистов.

**Функционал:**

Регистрация/Вход (Email, соцсети или телефон).

Личный кабинет пользователя.

История прослушиваний (нужна для алгоритма рекомендаций).

# Уровень 2: Функционал удержания и персонализации

## 5. Личные плейлисты (Private Playlists)

Пользователь должен иметь возможность сохранять музыку для себя.

**Функционал:**

Создание, редактирование (добавление/удаление треков), удаление личных плейлистов.

Сохранение состояния плейлиста в БД.

Доступ к плейлистам только для владельца.

## 6. Система тегирования и классификации

База для работы рекомендаций и поиска.

**Функционал:**

Глобальная система тегов: Жанры (Рок, Поп), Настроения (Грустное, Энергичное), Ситуации (Для спорта, Для сна).

Привязка каждого трека к нескольким тегам (осуществляется админом при загрузке).

Фильтрация каталога по этим тегам.

## 7. Алгоритм рекомендаций (Recommendation Engine)
Реализация пункта про «подбор в зависимости от предпочтений».

**Логика работы:**

Анализ истории прослушиваний пользователя.

Сопоставление любимых тегов пользователя с тегами в базе.

Классификация пользователей в группы («Любители рока», «Меланхолики») на основе их поведения.

Формирование ленты «Рекомендуем вам» или «Похоже на то, что вы слушали».

Уровень 3: Социальный функционал и Модерация

Функции для развития сообщества, требующие сложной логики проверки.

## 8. Публичные плейлисты и шеринг

Возможность делиться подборками с другими.

**Функционал:**

Конвертация личного плейлиста в статус «Публичный».

Генерация уникальной ссылки для шаринга.

Поиск публичных плейлистов по названию и тегам в общем каталоге.

Возможность подписаться на плейлист другого пользователя или скопировать его себе.

## 9. Система премодерации публичного контента

Критически важный блок безопасности и качества, учитывая ручное наполнение базы и риск спама.

**Процесс:**

При попытке опубликовать плейлист он переходит в статус «На проверке».

Автоматическая проверка (скрипты):

Проверка на дубликаты треков внутри одного плейлиста.

Проверка на наличие «мусорных» файлов (например, тишина, очень короткие сэмплы).

Валидность тегов (соответствуют ли теги плейлиста содержимому).

Ручная проверка (Админ/Модератор): Финальное утверждение перед публикацией.

Уведомление пользователя о статусе (Одобрено / Отклонено с причиной).


# Итоговая дорожная карта разработки

Этап 1: База данных + Админка (загрузка) + Плеер + Авторизация + Личные плейлисты.

Этап 2: Внедрение тегов + Алгоритм рекомендаций + Классификация пользователей.

Этап 3: Публичные плейлисты + Механизм модерации + Поиск по плейлистам.
